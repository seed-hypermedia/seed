# Frontend design

Inspect the existing design system, components, tokens, and nearby screens before choosing a direction. Existing product conventions take precedence unless the user explicitly requests a redesign.

For greenfield work, establish an intentional visual direction based on the interface's purpose, audience, and constraints. Make typography, color, spacing, composition, and motion reinforce that direction rather than applying a stock aesthetic.

Implement working, accessible code. Reuse project primitives and semantic tokens. Match the implementation complexity to the design: expressive concepts may justify richer motion or composition, while restrained designs require precise spacing and typography.

Avoid habitual defaults such as interchangeable dashboard layouts, decorative gradients with no relationship to the product, and gratuitous animation. Distinctiveness should come from the product context, not novelty for its own sake.

Verify responsive behavior, overflow, focus states, reduced-motion behavior, and consistency with adjacent UI before finishing.
